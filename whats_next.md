## Immediate Opportunities

### 1. Run Ralph Loop on a New Project
- Use `scripts/ralph-loop.sh` against a fresh TASKS.md to validate the one-task-per-iteration pattern
- Test with a different domain than fitness tracker to prove template flexibility
- Measure tokens per task, duration, and pass rate

### 2. Build Fitness Tracker Angular UI (Phase 2)

- Generate a completely new API (e.g., inventory tracking, incident management) using agent-one's runner

- Run L3 (compilation) and L4 (lint) tests on the output

- Finalize package.json for publishing
